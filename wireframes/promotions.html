Come funziona
Lo sconto appare come badge colorato sotto l'orario nel widget di prenotazione. Non modifica la caparra Stripe (che resta un deposito). Lo sconto si applica al conto al tavolo: il ristoratore lo comunica al cliente al momento del pagamento.
Pagina 2/3: Varianti per il badge sconto nel widget di prenotazione.
A) Badge esterno (posizionato in alto a destra dello slot) — piu visibile, stile TheFork.
B) Badge inline (sotto l'orario, dentro lo slot) — piu pulito, meno intrusivo.
Raccomandazione: Variante B (inline) per coerenza con il design minimale del widget.

Note tecniche di implementazione
Architettura e decisioni di design
Database
promotions
  id, tenant_id, name
  discount_percent (5-50)
  type: 'recurring' | 'specific_date'
  days_of_week (bitmask: 1=Lun...7=Dom)
  time_from, time_to (TIME, nullable)
  date_from, date_to (DATE)
  is_active (TINYINT, default 1)
  created_at, updated_at
API Availability (modifica)
GET /api/v1/tenants/{slug}/availability
  ?date=2026-03-24&party_size=2

Response slot aggiuntivo:
  "discount_percent": 20,
  "discount_name": "Martedi Sconto"

Se null → nessuno sconto per lo slot.
Flusso dati
  1. Ristoratore crea promozione nel dashboard
  2. API availability include discount_percent per slot applicabili
  3. Widget mostra badge arancione sugli slot con sconto
  4. Al submit, il widget invia discount_percent selezionato
  5. Backend salva su reservations.discount_percent
  6. Email + link magico mostrano lo sconto
  7. Lo sconto NON tocca Stripe/caparra
Regole di business
  • Max 1 promozione per slot (la piu alta vince)
  • Range sconto: 5% – 50% (step 5%)
  • Promozione senza date = sempre attiva
  • Toggle attiva/disattiva immediato
  • Sconto e informativo, il ristoratore lo applica al conto
  • Statistiche: conteggio prenotazioni per promozione
