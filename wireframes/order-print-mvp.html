MVP gratuito incluso in tutti i piani · Plugin "Stampa Pro" come addon a pagamento opzionale per esperienza tipo GloriaFood.

1 · Workflow stampa per status MVP gratuito

Il pulsante stampa cambia in base allo status dell'ordine. Su Accettati/In preparazione si stampa il ticket cucina (sintetico, solo per la brigata). Su Pronto si stampa la ricevuta completa cliente (con sezione staccabile firma). In tutti gli stati, finché il rider non è assegnato, compare il dropdown "+ Assegna rider".

Flusso operativo:
  1. Ordine arriva → ristoratore clicca Accetta → status diventa "Accettati"
  2. Ristoratore clicca 🍳 Stampa cucina → ticket sintetico parte verso la stampante della brigata. Niente prezzi, niente cliente, solo articoli + note
  3. (Nel frattempo assegna il rider con il dropdown — o lo farà dopo)
  4. Cucina lavora → ristoratore avanza a "In preparazione" → poi a "Pronto"
  5. Su Pronto clicca 🖨️ Stampa ordine → ricevuta completa per il cliente, con sezione staccabile firma in fondo
  6. Rider parte → al cliente lascia la metà superiore, riporta la metà firmata "Conferma consegna" → segna "Consegnato"

Sezioni della ricevuta MVP:

  • Header — nome ristorante + indirizzo + telefono (da tenant DB)
  • Order info — # ordine, data/ora ricevuto, pronto entro, pagamento
  • Cliente — nome, telefono, email
  • Indirizzo consegna (solo delivery) — via, CAP, note custom
  • Articoli — quantità, nome, varianti, modificatori, note
  • Totali — subtotale, consegna, totale grande
  • Footer "Visita il nostro sito" — usa tenants.website_url se popolato; fallback automatico a dash.evulery.it/{slug}. Riduce la richiesta di "dominio personalizzato" come feature
  • "Powered by Evulery" piccolo in fondo (branding discreto)
  • Linea staccabile ✂ + sezione "Conferma di consegna" — il rider taglia qui, fa firmare al cliente la metà sotto, riporta la firma al ristorante come prova consegna. Pattern visto in GloriaFood
  • QR "Stato ordine" rimandato: non esiste oggi una pagina pubblica di tracking ordine (orders manca manage_token). Da aggiungere quando facciamo il tracking pubblico (o nel plugin Print Pro)
Zero configurazione. Stesso layout per tutti i ristoratori. Funziona out-of-the-box appena attivata la migrazione. Cosmeticamente equivalente alla ricevuta GloriaFood ma senza l'editor — il ristoratore non perde tempo a configurare 15 toggle.

Privacy: il ticket cucina passa di mano in mano nella brigata e puo' restare appeso o caderne in giro. Nome cliente sì (utile per riconoscimento veloce "ordine di Mario Rossi"), telefono/email/indirizzo no: quelli vanno solo sulla ricevuta consegnata al rider.
⚠ Nota tecnica termica b/n: le stampanti termiche (Epson TM-m30III di Distinto Sushi incluse) sono bianco/nero pure. Sfondi colorati come il giallo vengono stampati come bianco — la testina termica si attiva solo per colori scuri. Per evidenziare le note usiamo reverse video (sfondo nero, testo bianco): la termica riempie l'area di punti neri e il testo resta in negativo. È lo standard ESC/POS, ben leggibile a distanza.

4 · Roadmap step-by-step
FASE 1 · MVP (~5-7h) · gratuito
Stampa HTML+CSS print, 2 template contestuali (questa sessione)
Approccio: HTML + CSS @media print, zero librerie PDF. CSS @page { size: 80mm auto; margin: 0 }, apertura nuova tab, Ctrl+P del browser. Funziona con qualsiasi stampante.

2 viste separate, 2 endpoint:
  • GET /dashboard/orders/{id}/print/kitchen → vista print-kitchen.php (ticket cucina sintetico, niente prezzi/cliente/indirizzo)
  • GET /dashboard/orders/{id}/print/receipt → vista print-receipt.php (ricevuta cliente completa con sezione staccabile firma)
Pulsanti contestuali in _card.php:
  • Su status accepted + preparing: pulsante "🍳 Stampa cucina"
  • Su status ready: pulsante "🖨️ Stampa ordine" (ricevuta cliente)
Cliente pilot Distinto Sushi: Epson TM-m30III via LAN + tablet Galaxy Tab A → Chrome Android via Mopria.
+ Migration tenants.website_url (✅ fatta ieri commit 6ce661a).
FASE 1.5 · TRACKING ORDINE (~2-3h) · gratuito
Pagina pubblica "a che punto è il mio ordine"
Migration orders.tracking_token, controller pubblico read-only /order/{token} con timeline stato (Ricevuto → Confermato → In preparazione → Pronto/In consegna → Consegnato), ETA stimato, contatti ristorante. Link incluso nell'email di conferma ordine.
Una volta pronto, il QR "Stato ordine" entra nella ricevuta MVP.
FASE 2 · Plugin Print Pro (futuro, ~15-20h)
Auto-print via PrintNode + ticket cucina
Integrazione PrintNode (servizio cloud print). Tenant configura API key + printer ID dalle settings. Ordini auto-stampati appena ricevuti. Aggiungiamo template "ticket cucina" senza prezzi. Trigger: lancio piattaforma + Distinto Sushi come pilot.
FASE 3 · Plugin Print Pro Plus (lontano, ~30h)
Editor template visuale + multi-stampante
Replica completa esperienza GloriaFood: editor visuale con toggle live, multi-template (cliente/cucina/quality check), multi-stampante per device, printing history 30gg con error tracking. Trigger: 5+ clienti che usano il plugin base e chiedono customization.
📝 Strategia plugin marketplace (note design decisions)
  • MVP gratuito incluso in tutti i piani: la stampa PDF è un'aspettativa di base nel 2026. Non un servizio premium.
  • Plugin a pagamento solo per automazione e customizzazione avanzata: il valore percepito è "tempo risparmiato all'arrivo dell'ordine" + "esperienza professionale tipo casse della grande distribuzione".
  • Naming plugin da definire: candidati "Evulery Print Pro", "Evulery Stampa Auto", "Print Cloud by Evulery". Decisione finale in sessione brand.
  • Marketplace pattern: questa è la PRIMA feature che diventa addon. Apre la strada a una sezione /dashboard/marketplace dove i ristoratori comprano funzionalità extra. Esempi futuri possibili: Loyalty Pro, Analytics Pro, Whatsapp Notifications, Riders App Companion.
  • Distinto Sushi come cliente pilot: migrazione da GloriaFood → Evulery + plugin Print Pro. Se accettano, validazione del modello commerciale + caso di studio per landing.
  • Pricing ipotetico: €19/mese per device (sotto GloriaFood ~€29) oppure setup fee one-shot €99 + €9/mese. Da validare con Distinto Sushi.
