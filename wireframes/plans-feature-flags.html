Gate UI - Come appare una feature bloccata
Statistiche Clienti

Le statistiche avanzate sono disponibili a partire dal piano Starter. Passa al piano superiore per sbloccare questa funzionalita.

// Controller - verificare accesso alla feature if (!PlanService::can('customer_stats')) { return view('dashboard/upgrade-required', [ 'feature' => 'Statistiche Clienti', 'min_plan' => 'starter', ]); } // View - mostrare gate inline <?php if (PlanService::can('export_csv')): ?> <a href="...">Esporta CSV</a> <?php else: ?> <span class="gate-locked">Piano Starter+ richiesto</span> <?php endif; ?> // Config: config/plans.php return [ 'free' => [ 'features' => ['widget', 'email_confirm'], 'limits' => ['reservations_monthly' => 50, 'users' => 1], ], 'starter' => [ 'features' => ['meal_categories', 'manage_link', 'manual_confirm', 'export_csv', 'customer_stats'], 'limits' => ['reservations_monthly' => null, 'users' => 3], ], 'pro' => [ 'features' => ['stripe_deposit', 'custom_domain', 'api_access'], 'limits' => ['reservations_monthly' => null, 'users' => null], ], ];

Note di Implementazione

  • Tabella DB: aggiungere colonna plan ENUM('free','starter','pro') DEFAULT 'free' a tenants
  • PlanService: classe statica, legge piano da TenantResolver::current()['plan'], confronta con config/plans.php
  • can($feature): controlla se la feature e nel piano corrente o superiore (ereditarieta: pro include tutto di starter)
  • limit($resource): ritorna il limite numerico (null = illimitato)
  • withinLimit($resource, $currentCount): ritorna bool per check contatore
  • Gate UI: componente riutilizzabile views/components/gate.php con parametri feature/min_plan
  • Sidebar: le voci di menu per feature bloccate mostrano lucchetto grigio, cliccabili ma portano alla pagina upgrade
  • Nessun pagamento in questa fase — solo logica di gating. Il pagamento verra aggiunto con Stripe in produzione
  • Admin panel: possibilita di cambiare piano manualmente per ogni tenant (utile per trial/demo)
